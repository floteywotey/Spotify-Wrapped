Inspired by tarot, we wanted to create a unique and fun way to access statistics about the things you listen to on Spotify, like Spotify Wrapped but can be accessed all year round! You can view your listening history from the past year, six months, or three months or even create duo wraps to compared your music taste with your friends. All the wraps that you create can be accessed in your past readings. We hope you enjoy!

User Stories

For this project, the main foundation of the project, where we first started, were targeting the base user stories such as:

  • Base User Story #1: As a user, I want to be able to view a presentation of the different aspects of my personal Spotify music listening tastes displayed in colorful and fun ways.
  • Base User Story #2: As a user, I would like to be able to create and login to an account that saves my previously generated Spotify wraps.
  • Base User Story #3: As a user, I would like the UI to be aesthetically-pleasing and responsive.
  • Base User Story #4: As a developer, I would like to avoid security leaks by not hosting secrets like API keys on GitHub.

Working on these base user stories was the main focus of Sprint 1. With the base of the project laid out, we were then able to implement additional user stories like:

  • Duo-Wrapped
  • Different Spotify Wrap durations
  • Dark/Light Mode
  • Running on a host service
  • Transition animations
  • Contact the developers
  • Wireframe diagram on Figma

During sprints 2 and 3, we worked on the additional user stories, primarily focusing on Duo-Wrapped, running on a hose service, transition animations, and contacting the developers (which you can find below).

Task Management

Notion was our main platform for organizing tasks within our team. Backend tasks were divided into specific functions such as filtering, password reset, and search function that were then assigned to each person. For frontend, tasks were divided into creating a UI flowchart, designing pages on Figma, and ultimately applying those designs with html and css. By subdividing tasks, placing them in “Not started”, “In progress”, and “Done”, and checking off a checklist of user stories in Notion, we were able to easily identify any tasks that needed to be picked up and which tasks were being implemented by someone else.

Scrum Meetings

Scrum meetings were conducted every weekend on Discord in order to check in with members in the team on what they had worked on for the week, decide what tasks would have to be prioritized and completed for the next week, and decide on what questions needed to be asked for our client meetings on Monday. Overall, we adapted based on feedback and issues encountered while developing the project.

Process Description

Brainstorm and References

While looking at pictures of old spotify wraps, we found that many ended by giving each person a unique label, such as time traveler and vampire. This sparked the idea of a tarot card which also had titles such as The Chariot or The Tower. We looked through the spotify API to find data we can pull to make these titles, originally using the track's unique features like danceability and positivity before they were removed.

UI Design-Frontend

We designed the potential pages and assets on Figma, allowing us to have a definitive plan for when we start implementing the front end and since we were integrating animations and targeting the user story for a fully developed wireframe, our goal was to make a high-fidelity prototype of the whole website. For assets and the tarot cards, we tried to combine the tarot theme and Spotify Wrapped features. For example, a crystal orb would ask the user about the duration of the Spotify Wrapped and each card would have a distinct design based on the title and Spotify Wrapped data. We detailed each presentation slide as a card, creating the backs, fronts, and how the information will be displayed for each card. For the other pages, we asked our client what information he’d like to have displayed. With each design, we made sure to check with our client for feedback.

The main focus while wireframing was making the presentation of the Spotify Wrapped unique. As designers without experience with tarot cards, we thought of ways that people hold and display cards. Therefore, based on our limited knowledge, we made different iterations for possible presentation of the Spotify Wrapped and received feedback from the client on their favorite design. We made sure that we made the transitions we wanted very clear so that incorporating it later won’t be as confusing.

When we got started with implementation, since the two front end developers had worked on different screens, we decided we would implement the screens we had designed. As the back end finished, we moved Kenny and John to implement some front end features.

Code - Backend

For the backend, we first established what points of data we needed to have for each page. After that was determined, we grabbed the data from the appropriate endpoints from the Spotify API. We also considered what kind of data we would need for different models, such as the user, the wraps, etc. We mostly focused on getting workable pieces of code out and testing it out through the app.
